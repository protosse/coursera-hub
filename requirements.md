# Coursera Hub 需求文档

## 1. 项目概述

### 1.1 项目名称

Coursera Hub

### 1.2 项目描述

Coursera Hub 是一个跨平台桌面客户端，用于下载 Coursera 网站的视频和课件资源。该应用提供友好的图形用户界面，支持多种认证方式和下载选项，让用户能够方便地批量下载已注册的课程内容。

### 1.3 目标用户

- Coursera 平台注册用户
- 需要离线观看课程视频的学习者
- 希望批量下载课程资源的教育工作者

---

## 2. 功能需求

### 2.1 用户认证

#### 2.1.1 认证方式

系统支持以下三种认证方式：

| 认证方式      | 描述                     | 输入参数          |
| ------------- | ------------------------ | ----------------- |
| CAUTH         | 直接输入 CAUTH Cookie 值 | CAUTH 字符串      |
| 浏览器 Cookie | 自动从浏览器提取 CAUTH   | 选择浏览器类型    |
| 用户名密码    | 使用 Coursera 账号登录   | 用户名/邮箱、密码 |

#### 2.1.2 支持的浏览器

- Chrome
- Chromium
- Opera
- Opera GX
- Brave
- Edge
- Vivaldi
- Firefox
- LibreWolf
- Safari

### 2.2 课程管理

#### 2.2.1 列出课程

- 显示用户已注册的所有课程列表
- 支持异步加载，实时显示日志输出
- 显示课程名称和相关信息

#### 2.2.2 课程下载

- 支持单个课程下载
- 支持专项课程（Specialization）下载
- 支持断点续传功能

### 2.3 下载配置

#### 2.3.1 下载选项

| 选项        | 描述                      | 默认值     |
| ----------- | ------------------------- | ---------- |
| 下载路径    | 课程保存位置              | 用户主目录 |
| 字幕语言1   | 主要字幕语言              | 英文       |
| 字幕语言2   | 备用字幕语言              | 中文       |
| 视频分辨率  | 视频清晰度                | 720p       |
| 下载测验    | 是否下载课程测验          | 是         |
| 下载笔记本  | 是否下载 Jupyter Notebook | 是         |
| 使用 aria2c | 是否使用 aria2c 加速下载  | 否         |

#### 2.3.2 下载器支持

- Native Downloader（Python 原生下载器）
- aria2c（多线程下载加速）
- curl
- wget
- axel

### 2.4 用户界面

#### 2.4.1 主要界面元素

1. **认证区域**
   - 认证方式选择（单选按钮组）
   - 认证信息输入框（根据认证方式动态显示）

2. **课程信息区域**
   - 课程名称输入框
   - 专项课程复选框
   - 使用 aria2c 复选框

3. **下载设置区域**
   - 下载路径输入框
   - 浏览按钮（选择目录）
   - 字幕语言选择（两个下拉框）

4. **操作按钮区域**
   - 列出课程
   - 下载课程
   - 恢复下载
   - 取消下载
   - 清空日志

5. **日志输出区域**
   - 滚动列表视图
   - 实时显示下载进度
   - 支持文本选择
   - 进度条实时更新（单行刷新）

#### 2.4.2 界面特性

- 响应式布局
- 浅色主题
- 窗口大小固定
- 边框和圆角美化

### 2.5 配置管理

#### 2.5.1 配置存储

- 配置文件路径：`~/.coursera_hub_config.json`
- 自动保存用户设置
- 应用启动时自动加载配置

#### 2.5.2 配置项

```json
{
  "auth_method": "认证方式",
  "cauth": "CAUTH值",
  "username": "用户名",
  "password": "密码",
  "course_name": "课程名称",
  "download_path": "下载路径",
  "subtitle_language1": "字幕语言1",
  "subtitle_language2": "字幕语言2",
  "is_special": "是否专项课程",
  "browser": "浏览器类型",
  "use_aria2": "是否使用aria2c"
}
```

### 2.6 日志系统

#### 2.6.1 日志功能

- 实时显示操作日志
- 捕获 print 输出
- 捕获 logging 输出
- 支持进度条显示（单行刷新）
- 移除 ANSI 转义序列
- 支持清空日志

#### 2.6.2 日志格式

- 纯文本格式
- 可选择复制
- 自动滚动到最新

## 3. 非功能需求

### 3.1 性能要求

- 异步执行长时间操作，不阻塞 UI
- 使用消息队列实现线程间通信
- 后台线程执行网络请求和下载任务

### 3.2 兼容性要求

- 操作系统：Windows、macOS、Linux
- Python 版本：3.8+
- 依赖框架：Flet

### 3.3 可用性要求

- 界面简洁直观
- 操作流程清晰
- 错误提示友好
- 配置自动保存

### 3.4 可维护性要求

- 代码结构清晰
- 模块化设计
- 使用枚举类型管理常量
- 遵循 Python 编码规范

## 5. 用户流程

### 5.1 认证流程

```
开始 → 选择认证方式 → 输入认证信息 → 验证认证 → 认证成功/失败
```

### 5.2 列出课程流程

```
开始 → 完成认证 → 点击"列出课程" → 显示课程列表 → 结束
```

### 5.3 下载课程流程

```
开始 → 完成认证 → 输入课程名称 → 设置下载选项 → 点击"下载课程" →
显示下载进度 → 下载完成/取消/失败 → 结束
```

## 6. 错误处理

### 6.1 认证错误

- CAUTH 值无效或过期
- 用户名或密码错误
- 浏览器 Cookie 访问权限问题（如 Safari）

### 6.2 下载错误

- 课程不存在
- 网络连接失败
- 磁盘空间不足
- 下载被用户取消

### 6.3 错误提示

- 友好的错误消息
- 日志窗口显示详细错误信息
- 提供可能的解决方案
